# CNN Feature Extractor and Embedding Analysis

This repository contains a Python application for extracting features from CNN models and visualizing their embeddings to explore semantic relationships. The tool supports pretrained models (VGG16, ResNet18, AlexNet) and custom models, with options for GPU acceleration and interactive GUI-based analysis.

## Features

* Extract features from images using pretrained or custom CNN models.
* Support for VGG16, ResNet18, and AlexNet with optional weight downloads.
* Custom model support via `.py` architecture files and `.pth` weight files.
* Preprocessing options: Resize, Crop, or None, with configurable dimensions for custom models.
* GPU acceleration using CuPy (if available) or CPU fallback.
* Interactive GUI with steps for model selection, architecture viewing, and embedding analysis.
* Visualization of embeddings using the `NpyVisualizerApp` with 2D/3D t-SNE and PCA/TruncatedSVD options.
* Logging and error handling for debugging.

## Requirements

* Python 3.8+
* Required libraries:
  * `torch`
  * `torchvision`
  * `pillow`
  * `numpy`
  * `cupy` (optional for GPU support)
  * `customtkinter` (optional for enhanced GUI; falls back to `tkinter`)
  * `matplotlib`
  * `seaborn`
  * `scikit-learn`

Install dependencies using:

```bash
pip install torch torchvision pillow numpy cupy-cudaXX matplotlib seaborn scikit-learn customtkinter
```

(Replace `cupy-cudaXX` with the appropriate CUDA version, e.g., `cupy-cuda11x`, or omit if using CPU only.)

## Usage

1. Run the application:
   ```bash
   python app.py
   ```
2. Follow the GUI steps:
   * **Step 1: Select Model** - Choose a pretrained model (VGG16, ResNet18, AlexNet) or load a custom `.py` file and `.pth` weights.
   * **Step 2: View Model Architecture** - Display available layers and their shapes, then select a layer for feature extraction.
   * **Step 3: Embedding Analysis** - Select dimension (2D/3D) and reduction algorithm (PCA/TruncatedSVD), then run the analysis.
3. Select an image folder to extract features, which are saved as `.npy` files and combined into an `.npz` file for visualization.

## File Structure

* `app.py`: Main application script with GUI and feature extraction logic.
* `visualisation_app.py`: Module containing the `NpyVisualizerApp` for embedding visualization.
* `features/`: Directory where extracted features are saved as `.npy` and `.npz` files.
* `app.log`: Log file for debugging (generated during runtime).
* `last_npz_path.txt`: Stores the path to the last saved `.npz` file.

## Output

* Features are saved as `.npy` files in the `features/<model_name>/<layer_name> (<transform>)/` directory.
* Combined features and labels are saved as `.npz` files in the same directory.
* Visualization plots and metadata are generated by `NpyVisualizerApp`.
